3. 35194d8 — Block stray Windows 'nul' files
--------------------------------------------
PowerShell treats `> nul` as a literal filename, not /dev/null, so
prior sessions accidentally created nul and docs/nul. Removed both
files (required [System.IO.File]::Delete with \\?\ prefix because
PowerShell special-cases the literal name even under -LiteralPath).
- .gitignore: added `nul` and `**/nul`.
- CLAUDE.md: added "Windows shell note" telling future sessions to
  use `| Out-Null` instead of `> nul`.
